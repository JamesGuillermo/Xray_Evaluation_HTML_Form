üìã OVERVIEW IN SIMPLE TERMS

Think of this system like a restaurant order system:

1. YOUR HTML FORM (index.html) = The Waiter
   - Takes your ratings and information
   - Packages everything neatly
   - Sends it to the kitchen

2. GOOGLE APPS SCRIPT (Code.gs) = The Kitchen
   - Receives the order (your ratings)

   - Records every submission
   - Keeps everything organized
   - Lets you calculate totals and analyze results

üîÑ THE COMPLETE WORKFLOW
================================================================================

STEP 1: User Fills Out the Form
-------------------------------
‚Üí Opens index.html in their browser
‚Üí Enters reviewer name, affiliation, date
‚Üí Rates each X-ray machine (0-5) for all 28 questions
‚Üí Clicks "Submit Evaluation"

STEP 2: Form Prepares the Data
-------------------------------
‚Üí JavaScript collects all the ratings
‚Üí Converts everything to numbers (no text mistakes!)
‚Üí Organizes data in JSON format like this:

  {
    "reviewer_name": "Dr. Smith",
    "affiliation": "City Hospital",
    "review_date": "2025-11-05",
    "ratings": {
      "q1_bullseye": 4,
      "q1_aimed": 3,
      "q1_ge": 5,
      "q2_bullseye": 3,
      ... (all 84 ratings - 28 questions √ó 3 machines)
    }
  }

STEP 3: Form Sends Data to Google Apps Script
----------------------------------------------
‚Üí Uses fetch() API to POST data to your Apps Script web app URL
‚Üí Sends it over HTTPS (secure connection)
‚Üí Waits for confirmation

STEP 4: Apps Script Receives and Processes
-------------------------------------------
‚Üí The doPost(e) function catches the incoming data
‚Üí Parses the JSON data
‚Üí Opens your specific Google Sheet
‚Üí Adds current timestamp
‚Üí Converts all ratings to pure numbers
‚Üí Appends a new row to the sheet

STEP 5: Confirmation Back to User
----------------------------------
‚Üí Apps Script sends back: {"status": "success"}
‚Üí Form shows green success message
‚Üí User sees: "‚úÖ Your evaluation has been saved to Google Sheets!"
