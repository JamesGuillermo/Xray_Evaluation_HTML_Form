-------------------------------------------

3. Right-click: "Windows PowerShell"

-------------------------------
1. In PowerShell (as administrator), type:

2. Press Enter

PART 9: TROUBLESHOOTING
================================================================================

ISSUE 1: "Command not found: firebase"
---------------------------------------
Symptom: PowerShell doesn't recognize "firebase" command
Solutions:
‚Üí Close and reopen PowerShell
‚Üí Reinstall Firebase CLI: npm install -g firebase-tools
‚Üí Check if installed: npm list -g firebase-tools
‚Üí Make sure Node.js is properly installed

ISSUE 2: "You must be logged in to access this resource"
---------------------------------------------------------
Symptom: Commands fail with authentication error
Solution:
‚Üí Type: firebase logout
‚Üí Type: firebase login
‚Üí Sign in again with your Google account

ISSUE 3: "Firebase hosting initialization has already been set up"
-------------------------------------------------------------------
Symptom: Can't run firebase init again
Solution:
‚Üí This is normal - you only init once
‚Üí To reconfigure, delete .firebaserc and firebase.json
‚Üí Then run: firebase init again

ISSUE 4: Deployment succeeds but site shows 404
------------------------------------------------
Symptom: Firebase URL shows "404 Not Found"
Causes & Solutions:

A. Wrong public directory:
   ‚Üí Open firebase.json
   ‚Üí Verify: "public": "."
   ‚Üí Redeploy: firebase deploy

B. index.html not in root:
   ‚Üí Verify index.html is in the same folder as firebase.json
   ‚Üí Use: dir (Windows) or ls (Mac/Linux) to check

C. Deployment didn't complete:
   ‚Üí Check for errors in deploy output
   ‚Üí Try: firebase deploy --debug for more info

ISSUE 5: Form loads but CSV doesn't load
-----------------------------------------
Symptom: Form shows "Loading questions..." forever
Solutions:
‚Üí Verify CSV file was deployed (check Firebase Console ‚Üí Hosting ‚Üí Files)
‚Üí Check CSV filename matches in index.html
‚Üí Clear browser cache (Ctrl+Shift+Del)
‚Üí Check browser console (F12) for CORS errors

ISSUE 6: Submissions don't reach Google Sheets
-----------------------------------------------
Symptom: Form submits but no data in sheet
Solutions:
‚Üí This is NOT a Firebase issue - it's Apps Script
‚Üí Follow CRITICAL_BUG_FOUND_AND_FIXED.txt
‚Üí Verify GOOGLE_SCRIPT_URL in index.html is correct
‚Üí Test Apps Script directly (visit URL in browser)
‚Üí Check Apps Script execution log

ISSUE 7: "Quota exceeded" error
--------------------------------
Symptom: Deployment fails with quota message
Solution:
‚Üí Usually means you're deploying too frequently
‚Üí Wait 30 minutes and try again
‚Üí Free tier: 10 deploys/hour limit

ISSUE 8: Changes not visible after deployment
----------------------------------------------
Symptom: Deployed changes but site looks the same
Solutions:
‚Üí Clear browser cache: Ctrl+Shift+R (hard refresh)
‚Üí Try incognito/private window
‚Üí Check Firebase Console ‚Üí Hosting ‚Üí Releases (verify deploy time)
‚Üí Wait a few minutes for CDN to update

ISSUE 9: SSL certificate pending
---------------------------------
Symptom: Custom domain shows "Certificate pending"
Solution:
‚Üí This is normal for first 24 hours
‚Üí Verify DNS records are correct
‚Üí Wait for automatic provisioning
‚Üí Check Firebase Console for status updates

ISSUE 10: "Firebase initialization failed"
-------------------------------------------
Symptom: firebase init shows errors
Solutions:
‚Üí Check internet connection
‚Üí Verify you're logged in: firebase login
‚Üí Make sure project exists in Firebase Console
‚Üí Try: firebase use --add (manually select project)
