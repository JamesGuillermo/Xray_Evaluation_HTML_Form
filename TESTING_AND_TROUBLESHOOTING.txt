6. COMMON ERROR MESSAGES & SOLUTIONS
================================================================================

ERROR: "Failed to load CSV file"
---------------------------------
Symptoms:
- Red error box on page load
- Form doesn't appear
- Console shows fetch error

Causes & Solutions:

1. CORS Issue (most common)
   Symptom: "CORS policy: No 'Access-Control-Allow-Origin' header"
   Solution: Use a web server (http://), not file:// protocol
   ‚Üí Run start_server.bat (Windows)
   ‚Üí Or use: python -m http.server 8000
   ‚Üí Or use index_offline.html with file upload

2. File Not Found
   Symptom: "404 Not Found"
   Solution: Check file exists in same folder as index.html
   ‚Üí Verify filename matches CSV_FILE_NAME constant
   ‚Üí Check for typos in filename

3. Wrong Path
   Symptom: File exists but still not found
   Solution: Ensure CSV is in SAME directory as index.html
   ‚Üí Not in a subfolder
   ‚Üí Not in parent folder

4. File Permissions
   Symptom: "Permission denied"
   Solution: Check file read permissions
   ‚Üí Right-click file ‚Üí Properties ‚Üí Security
   ‚Üí Ensure "Read" is allowed

---

ERROR: "‚ùå Error Saving to Google Sheets"
------------------------------------------
Symptoms:
- Form submits but shows error alert
- Data doesn't appear in Google Sheet
- Console shows network error

Causes & Solutions:

1. Invalid Web App URL
   Symptom: 404 error in console
   Solution: Verify GOOGLE_SCRIPT_URL in index.html
   ‚Üí Must end with "/exec"
   ‚Üí Copy from Apps Script deployment page
   ‚Üí No trailing spaces

2. Apps Script Not Deployed
   Symptom: "Authorization needed" or redirect to login
   Solution: Deploy (or re-deploy) Apps Script as web app
   ‚Üí Apps Script ‚Üí Deploy ‚Üí New deployment
   ‚Üí Type: Web app
   ‚Üí Access: Anyone
   ‚Üí Copy new URL

3. Wrong Permissions
   Symptom: "You need permission" message
   Solution: Change web app access to "Anyone"
   ‚Üí Apps Script ‚Üí Deploy ‚Üí Manage deployments
   ‚Üí Click edit (pencil icon)
   ‚Üí Who has access: Anyone
   ‚Üí Update

4. Network Connectivity
   Symptom: "Failed to fetch" or timeout
   Solution: Check internet connection
   ‚Üí Try opening script.google.com in browser
   ‚Üí Check firewall settings
   ‚Üí Try different network

5. Script Runtime Error
   Symptom: Returns {status: 'error'} with message
   Solution: Check Apps Script execution log
   ‚Üí Apps Script ‚Üí View ‚Üí Execution log
   ‚Üí Look for red error messages
   ‚Üí Common: Wrong SHEET_ID, sheet not found

---

ERROR: "Missing required fields"
---------------------------------
Symptom: Can't submit even with all fields filled
Cause: Validation logic issue

Solutions:
1. Check browser console for JavaScript errors
2. Verify all input fields have correct 'name' attributes
3. Test in different browser
4. Clear browser cache and reload

---

ERROR: #REF! in Google Sheet totals
------------------------------------
Symptom: Total columns show #REF! instead of numbers
Cause: Formula references non-existent cells

Solutions:
1. Check formula references correct column range
2. Verify all rating columns exist (E through CL)
3. Delete row and let Code.gs recreate with correct formulas
4. Manually fix formula: =SUMPRODUCT((MOD(COLUMN(E2:CJ2)-COLUMN(E2),3)=0)*(E2:CJ2))

---

ERROR: Data in wrong columns
-----------------------------
Symptom: Ratings appear shifted or in wrong order
Cause: Header row doesn't match Code.gs expectations

Solutions:
1. Delete all rows including header
2. Submit a test - Code.gs will recreate proper headers
3. Or manually verify headers match GOOGLE_SHEET_SETUP.txt exactly
4. Check for extra spaces in header names

9. DATA INTEGRITY CHECKS
================================================================================

After collecting real data, periodically verify integrity:

CHECK 1: No Missing Timestamps
-------------------------------
In Google Sheet:
1. Select column A (Timestamp)
2. Look for any blank cells
3. If found: Data submission failed for that row

CHECK 2: All Ratings Are Numbers 0-5
-------------------------------------
In Google Sheet:
1. Select rating columns (E through CL)
2. Data ‚Üí Data validation ‚Üí Show warning (already set)
3. Look for any cells with text or numbers > 5
4. If found: Validation bypassed somehow

CHECK 3: Totals Calculate Correctly
------------------------------------
Pick a random row and manually verify:
1. Note the totals in columns CM, CN, CO
2. Use calculator to sum the corresponding ratings
3. Should match exactly

Example:
If BullsEye Total (CM) shows 87:
- Sum columns E, H, K, N, Q, T, W, Z, AC, AF, AI, AL, AO, AR, AU, AX, BA, BD, BG, BJ, BM, BP, BS, BV, BY, CB, CE, CH
- Should equal 87

CHECK 4: No Duplicate Submissions
----------------------------------
Look for:
- Same reviewer name
- Same timestamp (within seconds)
- Identical ratings

If found:
- May indicate double-submission bug
- Or two people with same name (rare but possible)

CHECK 5: Reasonable Rating Distribution
----------------------------------------
Statistical check:
- If 90%+ of all ratings are 0 or 5: Suspicious (not using scale properly)
- If all ratings identical across rows: Copy-paste error
- Expected: Bell curve distribution with most ratings 2-4

10. ROLLBACK PROCEDURES
================================================================================

If something goes seriously wrong:

SCENARIO 1: Bad Data in Google Sheet
-------------------------------------
Solution: Delete bad rows
1. Identify affected rows
2. Right-click row number ‚Üí Delete row
3. Or use clearAllResponses() function in Code.gs (deletes ALL data)

SCENARIO 2: Apps Script Broken
-------------------------------
Solution: Restore previous version
1. Apps Script ‚Üí File ‚Üí Version history
2. Select working version
3. Click "Restore"
4. Re-deploy as web app

SCENARIO 3: Form Not Working
-----------------------------
Solution: Use previous index.html version
1. If using version control: git checkout previous version
2. If not: Keep backup copies of working files
3. Replace current index.html with backup

SCENARIO 4: Lost Submissions
-----------------------------
Prevention: Enable email notifications
1. In Code.gs: SEND_EMAIL_NOTIFICATIONS = true
2. Each submission sends confirmation email
3. Emails serve as backup

Recovery:
- Check browser console logs (may have submitted data)
- Check Apps Script execution log
- Ask users to resubmit if necessary

SCENARIO 5: Google Sheet Accidentally Deleted
----------------------------------------------
Recovery:
1. Google Drive ‚Üí Trash
2. Find deleted sheet
3. Right-click ‚Üí Restore
4. Verify SHEET_ID still matches Code.gs

SCENARIO 6: Need to Start Over
-------------------------------
Complete reset:
1. Delete all rows in Google Sheet (keep header)
2. Or create new sheet tab and update SHEET_NAME
3. Clear browser cache (Ctrl+Shift+Delete)
4. Test with fresh submission
